StarRez — Residential Community Management Platform
Vendor: StarRez, Inc.

Executive Summary

StarRez is a cloud-based residential community management platform widely adopted by universities, student housing operators, and purpose-built student accommodation (PBSA) providers globally, including in Australia. It offers a comprehensive REST API (v2.0) with full CRUD operations and a proprietary query language (StarQL), a certified Microsoft Power Automate connector with 33 actions, and native integrations with over 40 technology partners spanning payment processing, access control, SIS platforms, maintenance, and communications. No Zapier, Make, or MCP integrations have been identified. A mobile app (StarRez Go / StarRez Housing) is available on both iOS and Android for operational staff.

System Overview

StarRez is a cloud-based platform designed to manage residential communities, with a primary focus on student housing, purpose-built student accommodation (PBSA), and multi-family residential operations. The platform covers the full resident lifecycle including applications, room allocation, contracts, billing, maintenance requests, inspections, and resident communications. It also includes a resident-facing self-service portal (PortalX) for room selection, payments, and service requests.

StarRez operates as a Software-as-a-Service (SaaS) platform hosted in the cloud, with instances deployed globally. It occupies a leading position in the higher education and PBSA property management market, serving over 1,300 institutions and operators worldwide. The platform is used by major Australian universities and PBSA operators, and supports integration with Australian payment providers including NAB, BPOINT, and Westpac PayWay.

Building Technology Relevance

StarRez is not a building management system (BMS) or building automation system (BAS), and it does not communicate via BACnet, Modbus, OPC-UA, or MQTT protocols. However, it sits adjacent to building technology systems in residential and campus environments through its integrations with access control (ASSA ABLOY, RemoteLock, CBORD, Onity), maintenance management (Archibus, TMA Systems, EZ Turn), and facility scheduling (EMS). In smart building contexts, StarRez is relevant as the property/tenancy management layer that feeds occupancy and resident data to downstream building systems via its REST API.

B. API Integration

StarRez provides a comprehensive REST API (version 2.0) that allows external systems to read, write, and manage data across all major database tables. The API supports full CRUD operations, ad-hoc querying via a proprietary query language (StarQL), business function execution, attachment and photo management, account transactions, report retrieval, and student authentication including magic link generation. Authentication is via web service tokens generated within the StarRez admin interface.

API Type: RESTful HTTP (supports JSON and XML responses)
Authentication: Web Service Token (username + token generated in StarRez Account settings)
Documentation URL: StarRez REST 2.0 API Guide (StarCare login required)

Key API Characteristics

  • StarQL Query Language: Proprietary ad-hoc query language for complex data retrieval via the /query endpoint
  • Schema Introspection: Full database schema discovery (tables, columns, relationships, functions) via the /databaseinfo endpoints
  • Business Functions: Execute workflow operations (check-in, check-out, etc.) programmatically via /function endpoint
  • Staff Mobile API: Dedicated V2 endpoints for mobile app operations including check-in/out, inspections, inventory scanning
  • Rate Limit: 100 API calls per connection per 60 seconds (as documented for the Power Automate connector)

Recommendations

  1. Leverage the REST API for building technology integration. StarRez's comprehensive API can feed occupancy, booking, and resident data to BMS/BAS systems, access control platforms, and smart building dashboards via middleware or API gateway.
  2. Use Power Automate for workflow automation. The certified Premium connector covers core operations; for advanced queries, pair it with the HTTP Request connector using the full REST API and StarQL.
  3. Explore custom MCP server development. Given the REST API's self-describing schema endpoints, building an MCP server for StarRez data would be feasible and could enable AI-powered querying of housing data.
  4. Consider API-based integration for access control. If using access control systems not covered by native partners (e.g. Gallagher, Salto), the REST API provides the data layer for custom integration.
  5. Note the Australian payment provider support. NAB, BPOINT, and Westpac PayWay are natively supported, which is relevant for Australian PBSA operations.
