Integration Capability Report

Salto Space — Access Control Platform
Vendor: SALTO Systems
Date: 30 March 2026
Prepared by: Meld Strategies

Executive Summary

This report documents the integration capabilities of Salto Space, SALTO Systems' on-premise access control platform, as researched for Meld Strategies on 30 March 2026. The research draws on SALTO's official developer portal, support documentation, and certified technology partner resources.

Salto Space offers a proprietary integration framework centred on the SALTO Host Interface Protocol (SHIP), which supports bidirectional data exchange with CCTV, fire alarm, building automation, and time-and-attendance systems. A dedicated Hospitality REST API (v6.10+) enables Apple Wallet digital key management for hospitality environments. Native integrations exist with leading physical security and PSIM platforms including Genetec Security Center, LenelS2 OnGuard, Johnson Controls C•CURE 9000, Siemens SiPass, and ICT Protege GX.

No MCP (Model Context Protocol) server or direct no-code automation connectors (Zapier/Make) were identified for Salto Space. The platform is on-premise by design, and third-party cloud integrations are facilitated via the Seam API bridge. Mobile access is supported through the Salto JustIN Mobile App on iOS and Android.

Recommended next steps: Engage SALTO's technical team to obtain the full SHIP specification (NDA required) and evaluate the Seam API bridge as a pathway to cloud-based access control integration for future smart building projects.

System Overview

Salto Space (also referred to as ProAccess SPACE) is SALTO Systems' on-premise access control management platform. It is an industry-leading smart access control solution built around SALTO's proprietary SVN (SALTO Virtual Network) data-on-card technology, enabling wire-free, keyless electronic locking across up to 64,000 doors and 4,000,000 users. The platform manages access-time zones, audit trails, cardholder permissions, visitor management, lift/elevator access, and locker control through a web-based interface.

Salto Space is deployed locally on Windows-based server hardware and operates over standard Ethernet IP networks. It is distinguished from SALTO's cloud-based platform (Salto KS) by its emphasis on offline resilience, compliance-grade audit trails, and on-premise data sovereignty — making it well-suited to regulated environments such as healthcare, education, government, and large commercial office projects. The platform supports both offline (data-on-card SVN) and online (BLUEnet wireless) lock architectures, as well as credentials spanning RFID, NFC, BLE, and mobile keys.

Building Technology Relevance

Salto Space is a core access control system directly relevant to BMS/BAS, PSIM, and smart building integration contexts. It supports building automation system (BAS) integration via a licensed BAS protocol module, and SHIP protocol integration with CCTV, fire alarm, and time-and-attendance platforms. It is a standard access control specification item for commercial office, hospitality, healthcare, education, and mixed-use developments in the Australian market.

A. Native Integrations

Salto Space integrates with third-party platforms primarily through the proprietary SALTO Host Interface Protocol (SHIP), the SALTO Virtual Network (SVN) plugin, and a licensed BAS (Building Automation System) protocol module. SHIP enables bidirectional communication with security management, CCTV, fire alarm, time-and-attendance, and building management systems over standard TCP/IP. The BAS module supports integration with in-room control systems using wireless protocols including BLUEnet (Bluetooth LE), RFnet, and custom IEEE 802.15.4 (ZigBee) profiles. Third-party card readers are integrated via Wiegand interface (26, 32, 34, 35, or 37-bit formats).

B. API Integration

Salto Space provides two API pathways for integration: the proprietary SALTO Host Interface Protocol (SHIP) and a purpose-built REST API (Hospitality API) introduced in Space v6.10.

B1 — SALTO Host Interface Protocol (SHIP)

SHIP is SALTO's proprietary bidirectional protocol for integrating Salto Space with third-party security and building management systems (CCTV, fire alarm, time and attendance, BMS). It operates over TCP/IP using either the Salto Transport Protocol (STP) or HTTP/S with Basic or Custom token authentication. SHIP access requires an NDA with SALTO and is licence-dependent. The SHIP specification document is available under NDA from SALTO technical support. Salto Space can operate as both a SHIP server (exposing data to a third-party host) and a SHIP client (pushing data to a third-party server).

API Type: Proprietary SHIP (TCP/IP — STP or HTTP/S)

B2 — Hospitality REST API

From Space v6.10+, a RESTful HTTP endpoint is available for hospitality management system integration, specifically for managing Apple Wallet digital room keys. This API is designed for hotel environments where a credential manager (e.g. Duve, Flexipass, MEWS) communicates with Salto Space to issue, manage, and revoke guest wallet keys. Authentication is via Bearer token (issued by SALTO). The API is documented on the SALTO Developer Portal with an OpenAPI (YAML) specification available for download. This API is licence-dependent (Apple Wallet key functionality requires a separate licence).

API Type: RESTful HTTP (OpenAPI/YAML specification available)

B3 — Seam API Bridge (Third-Party)

For organisations wishing to integrate Salto Space with cloud-based platforms, the Seam API (third-party) provides a bridge via the Seam Bridge software component. Seam Bridge is installed on the same Windows computer running Salto Space and connects to the Seam cloud platform, exposing access control management capabilities via Seam's unified API. This enables access system user management, access group assignment, entrance monitoring, and credential management (key cards and mobile keys) through a cloud-hosted RESTful API.

Documentation: docs.seam.co — Salto ProAccess Space

No MCP (Model Context Protocol) server has been identified for Salto Space at the time of this report. SALTO Systems has not published an MCP server for either Salto Space or Salto KS. D. No-Code Automation Integrations

No native no-code automation integrations (Zapier, Make, n8n, Power Automate) have been identified for Salto Space (the on-premise platform). This is consistent with Salto Space's on-premise, network-isolated architecture. SALTO's cloud platform, Salto KS, does support broader integration with cloud-first tools and is designed for organisations requiring automation and multi-site scalability. If no-code automation is a project requirement, the Seam API bridge may provide an indirect pathway, subject to Seam's own Zapier/Make connector availability.

E. Mobile App

SALTO provides the JustIN Mobile App as the mobile key solution for Salto Space environments. The app allows users to use their smartphone as a digital access credential, replacing physical key cards. JustIN Mobile operates over Bluetooth Low Energy (BLE) and supports both near detection mode (up to 30 cm) and far opening mode (up to 15 m) at compatible online readers. Keys are assigned to user phone numbers by Space system administrators and distributed securely. The app also includes a JustIN Mobile SDK for third-party integrations.

Recommendations & Next Steps

  1. Obtain the SHIP specification under NDA — Engage SALTO's technical or channel team to execute the required NDA and obtain the full SHIP protocol specification for any project requiring integration with PSIM, CCTV, BMS, or time-and-attendance platforms.
  2. Confirm BAS licence requirements early — BAS integration (for in-room control systems such as Inncom/Honeywell) is licence-dependent; confirm this during design phase to avoid procurement delays.
  3. Evaluate Seam API for cloud integration projects — Where cloud-based access control management or API-driven integration is required, assess the Seam Bridge solution as a supported third-party pathway for Salto Space.
  4. Specify Salto Space v6.10+ for hospitality projects — Ensure the correct software version is specified for any hospitality project requiring Apple Wallet key integration via the Hospitality REST API.
  5. Consider Salto KS for multi-site or automation-heavy projects — Where no-code automation, cloud management, or Zapier/Make workflows are required, Salto KS (cloud) may be more appropriate and should be evaluated as an alternative.
